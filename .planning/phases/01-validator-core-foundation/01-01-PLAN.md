ValidationResult (locked by user decision):
```typescript
export interface ValidationResult {
  email: string;
  status: 'valid' | 'invalid' | 'risky';
  score: number;          // 0 (valid) | 50 (risky/catch-all) | 100 (invalid)
  reason: string | null;  // null for valid; 'syntax'|'no-domain'|'no-mx'|'disposable'|'role'|'catch-all'|'typo'
  suggestion?: string;    // only when reason === 'typo', full corrected email e.g. 'user@gmail.com'
}
```

Pipeline order (fail-fast, from RESEARCH.md):
1. syntax    → invalid, score 100, reason 'syntax'
2. role      → invalid, score 100, reason 'role'
3. disposable → invalid, score 100, reason 'disposable'
4. typo      → invalid, score 100, reason 'typo', suggestion: corrected full email
5. catchall curated list → risky, score 50, reason 'catch-all'
6. MX lookup (async DoH) → 'no-domain' | 'no-mx'
7. MX heuristic (own MX server) → risky, score 50, reason 'catch-all'
8. All clear → valid, score 0, reason null

DoH endpoint: https://cloudflare-dns.com/dns-query?name={domain}&type=MX
Accept header: application/dns-json
Response: { Status: 0 (ok) | 3 (NXDOMAIN), Answer?: [{ data: "10 mx.host.com." }] }
MX data field parsing: split on first space → take [1] → strip trailing dot → lowercase

Shared MX providers (NOT catch-all):
google.com, googlemail.com, outlook.com, hotmail.com, protection.outlook.com,
yahoo.com, yahoodns.net, protonmail.ch, proton.me, icloud.com, apple.com,
zoho.com, amazonses.com, sendgrid.net, mailgun.org, mimecast.com, pphosted.com
